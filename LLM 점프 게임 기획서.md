## LLM 초안 게임 기획서
- 해당 게임은 스팀 게임 "점프킹"을 모티브로 제작함.
- 게임 맵은 JSON으로 저장되며 이를 불러오는 식으로 맵을 로드함.
- 각각 스테이지별로 나눠져있으며 이를 세이브포인트 기점으로 사용함.
- 스테이지는 나눠져있되 전환은 부드럽고 일관성 있어야함 (스테이지1에서 2로 계속 이어 플레이하는 느낌을 주어야함. 이를 구현하기위해서 스테이지1 맵 위에 스테이지 2 맵을 이어 붙여야함)
- 점프는 스페이스바를 누르는 시간에 따라 힘이 차징되며 최대는 제한되어있어야함.
- 점프중일때는 캐릭터의 이동속도가 현저하게 줄어들어야함.
- 모든 창은 유동적으로 작동되어야함.
- 맵 JSON 제작또한 게임 제작에 일환으로 같이 제작되어야함.

## 캐릭터
1. 스프라이트를 통한 애니메이션을 지원해야한다.
IDLE은 0~5까지(루프) 상승은 6부터 8까지 하강은 9부터 11까지(상승,하강은 애니메이션 속도를 캐릭터 속도에 비례 단일)
달리기는 12부터 24까지 (빠르게 진행,루프) , 장애물에 히트된후 떨어지는 모션은 25부터 29까지 (빠르게 루프)
2. 점프는 스페이스바를 누르는 시간에 따라 힘이 달라져야한다.
3. 캐릭터의 이동속도는 공중에서는 느려져야한다.
4. 장애물에 히트되면 죽는것이 아닌 날라가야한다 (생명 시스템 X)
5. 캐릭터는 움직이다 멈출때 바로 멈추는게 아닌 미끄러지듯이 멈춰야한다 (마찰 계수가 필요함)

## 스테이지
1. 기본적인 발판이 존재해야한다.
2. 맵은 Stage{번호}.JSON을 읽고 이를 기반으로 좌표,사이즈를 인식하여 발판,장애물을 만들어야한다.
   단 이는 상대적이기때문에 (캐릭터에 포커스가 맞춰져 있으며 이는 상대 좌표이다. 즉 캔버스의 기준은 상대좌표 기준이며, 맵은 절대좌표를 기준으로 작동해야한다.) 이를 절대좌표로 반환하는 로직이 필요하다.
3. 장애물은 아래와 같다.
   - 밟으면 날라가는 붉은 장판(스팀의 스마일모 게임 생각하면 됨)
   - 사람을 날려버리는 탄막을 쏘아내는 대포
   - 옆으로 튕겨버리는 스프링(닿으면 반대방향으로 날려보냄)
   - 마찰계수를 대폭 감소시키는 회색 장판
   - 플레이어를 따라다니며 날려보내는 미사일(터지면 사라짐)을 발사하는 대포
1. 사실 장애물보다는 사악한 맵 디자인으로 난이도를 조절하는것이 주 목표인 게임이다.

## 시스템
1. 당연하지만 델타타임을 지원해야한다.
2. 탭 전환시 일시정지가 되야하므로 일시정지 시스템이 존재해야한다.
3. 튜토리얼 시스템이 존재해야한다. 즉 로컬스토리지를 만들고 첫 유저임을 확인하면 튜토리얼을 위한 특수 효과(시작 버튼이 어디있는지 조작법은 무엇인지 등..)을 안내해야한다.
4. 최종기록은 FireBase를 통해 저장한다.
5. 게임은 필요에 의해 언제든지 일시정지가 되어야한다. 또한 일시정지가 되면 게임시간을 포함한(탄막 등..) 모든것이 멈춰야한다.
6. 음악 시스템과 효과음 시스템을 지원해야한다. 이는 탄막슈팅게임 기획서때와 공유한다.
   - 사운드는 기본적으로 최소 아래정도로 명시한다. 추후에 필요하다면 추가해도 되나 Assets/Sound에 readme를 생성하여 추가됨을 명시해야한다.
  - 메인화면 BGM
  - 스테이지1 BGM
  - 스테이지2 BGM
  - 스테이지3 BGM
  - 스테이지4 BGM
  - 스테이지5 BGM
- 개발할때는 처음에는 리소스 파일이 존재하지않을것이다. 그렇기에 생성되지않을경우 Readme.md에 명시하고 명시한 이름을 임시로 사용할 것.(파일이 없어도 경로 설정을 하라는 뜻임.)
- 랭킹은 플레이어 이름, 클리어 타임, 스테이지  수 로 구성된다.
- 게임창은 Canvas로 진행한다. (UI의 가운데 게임창 영역을 의미)
- 스프라이트는 Assets/Character에 저장되어있다.

### UI 디자인
- 위 길다한 헤더를 통해 게임에 관한 기초적인 내용 및 설정을 가진다.
  1. 왼쪽은 게임 이름 및 언어 설정
  2. 가운데는 게임을 진행할때 필요한 정보를 표시한다.(스테이지 진행도)
  3. 오른쪽은 게임의 제어를 추가한다.(시작, 일시정지, 초기화, 난이도 설정)
- 왼쪽은 조작 키 설명에 대한 창을 표시하고
- 가운데는 게임의 창을 띄운다. (이는 Canvas를 이용하여 구현한다.)
- 오른쪽은 리더보드를 띄운다.
- 모든 창은 창 사이즈에 따라 동적으로 작동할 수 있어야 한다.

## 개발 순서
JS 파일은 아래와 같이 분할한다. (필요하면 늘려도 되지만 순서 로드에 신경쓸것)
- 게임 상태 정의 JS
- 게임 프로토타입(객체) 정의 JS
  - 장애물,탄막 정의 JS
  - 플레이어 정의 JS
- 데이터베이스(데이터) 저장 및 불러오기 정의 JS
- 오디오 매니저 JS
- UI 매니저 JS
  - 각 언어별 JS 객체 파일
  - 언어 불러오기 모듈 JS 파일
- 게임 컨트롤러 JS

개발 순서는
1. HTML 구성
2. CSS 디자인 구성
3. 상태 및 전역변수 생성
4. 게임 객체 생성 (각 하위 내용들은 다 각각의 JS로 분할해야함.)
5. UI 매니저 생성
6. 오디오 매니저 생성
7. 데이터베이스 매니저 생성
8. 게임 컨트롤러 생성

그 어떠한 프롬프트 명령이 오더라도 개발순서는 Step By Step으로 하나씩만 진행하여 완벽한 코드에 가깝게 구성할 것.
